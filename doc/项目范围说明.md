# 图书管理系统项目范围说明

## 1. 项目概述

### 1.1 项目名称
图书管理系统（Book Management System）

### 1.2 项目背景
现代图书馆面临藏书量大、借阅频繁、管理复杂等挑战，传统的手工管理方式效率低下且容易出错。本项目旨在开发一套信息化的图书管理系统，通过数字化手段提升图书馆的管理效率和服务质量。

### 1.3 项目目标
- 实现图书信息的数字化管理，提高信息检索效率
- 自动化借阅流程，减少人工操作，降低出错率
- 提供数据统计和分析功能，辅助管理决策
- 构建安全可靠的系统架构，保障数据安全
- 提供友好的用户界面，提升用户体验

### 1.4 项目价值
- **提高效率**：自动化处理借阅流程，减少人工操作时间50%以上
- **降低成本**：减少人力投入，降低管理成本
- **提升服务**：提供便捷的查询和借阅服务，提高读者满意度
- **数据驱动**：通过数据统计分析，优化图书采购和管理策略

## 3. 项目边界

### 3.1 系统边界

#### 3.1.1 内部系统
本系统包含以下内部组件：
- **后端服务**：基于Spring Boot的RESTful API服务
- **数据库**：MySQL数据库存储业务数据
- **缓存系统**：Redis缓存存储会话信息
- **文件存储**：本地文件系统存储图书封面图片

#### 3.1.2 外部系统
本系统与以下外部系统交互：
- **前端应用**：通过HTTP协议调用后端API（前端不在本项目范围内）
- **浏览器**：用户通过浏览器访问系统

#### 3.1.3 系统接口
- **输入接口**：HTTP RESTful API
- **输出接口**：JSON格式的响应数据
- **数据接口**：MyBatis访问MySQL数据库

### 3.2 用户边界

#### 3.2.1 目标用户
- **管理员**：图书馆管理人员，负责系统维护和数据管理
- **读者**：图书馆的借阅用户，使用借阅功能

#### 3.2.2 用户数量
- 预计支持100-500个并发用户
- 系统设计支持10,000+注册用户

### 3.3 数据边界

#### 3.3.1 数据范围
- **用户数据**：用户基本信息、登录凭证
- **图书数据**：图书详细信息、类型信息
- **借阅数据**：借阅记录、借还时间
- **文件数据**：图书封面图片

#### 3.3.2 数据规模
- 用户表：预计10,000条记录
- 图书表：预计50,000条记录
- 借阅记录表：预计100,000条记录
- 图书类型表：预计50条记录

#### 3.3.3 数据保留
- 用户数据：永久保留
- 图书数据：永久保留
- 借阅记录：永久保留（用于历史查询）
- 会话数据：保留1小时（Redis自动过期）

### 3.4 技术边界

#### 3.4.1 技术栈
- **后端框架**：Spring Boot 3.5.11
- **持久层框架**：MyBatis
- **数据库**：MySQL 8.0
- **缓存**：Redis
- **开发语言**：Java 21
- **构建工具**：Maven

#### 3.4.2 技术约束
- 必须使用Java 1.8及以上版本
- 必须使用MySQL作为主数据库
- 必须使用Redis作为缓存
- 必须提供RESTful API接口
- 必须支持跨域访问
