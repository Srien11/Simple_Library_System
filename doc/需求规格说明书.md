# 图书管理系统需求规格说明书

## 1. 引言

### 1.1 编写目的
本文档旨在详细描述图书管理系统的功能需求、性能需求和设计约束，为系统开发、测试和验收提供依据。

### 1.2 项目背景
随着图书馆藏书量的不断增加，传统的手工管理方式已无法满足现代化管理的需求。本系统旨在通过信息化手段，实现图书信息的数字化管理、借阅流程的自动化处理，提高图书馆的管理效率和服务质量。

### 1.3 术语定义
- **管理员**：具有系统完全访问权限的用户，负责图书信息维护、用户管理等工作
- **读者**：普通用户，可以浏览图书信息、借阅和归还图书
- **图书状态**：图书的借阅状态，包括"可借"和"已借出"两种
- **借阅记录**：记录读者借阅图书的详细信息，包括借阅时间和归还时间

### 1.4 参考资料
- 《软件工程：实践者的研究方法》
- 《图书馆管理系统业务流程规范》
- RESTful API 设计规范

## 2. 系统概述

### 2.1 系统目标
构建一个功能完善、操作便捷、安全可靠的图书管理系统，实现以下目标：
- 实现图书信息的数字化管理
- 简化借阅流程，提高借阅效率
- 提供多维度的数据查询和统计功能
- 保证数据的安全性和一致性
- 支持多用户并发访问

### 2.2 系统范围
本系统覆盖图书馆日常管理的核心业务流程，包括：
- 用户管理（注册、登录、权限控制）
- 图书信息管理（增删改查、分类管理）
- 借阅管理（借书、还书、记录查询）
- 数据统计与查询

### 2.3 用户特点
- **管理员用户**：熟悉图书馆业务流程，具备基本的计算机操作能力
- **读者用户**：年龄跨度大，计算机操作水平参差不齐，需要简洁直观的操作界面

## 3. 功能需求

### 3.1 用户管理模块

#### 3.1.1 用户注册
**功能描述**：新用户可以通过注册功能创建账号。

**输入**：
- 用户名（必填，唯一）
- 密码（必填，长度6-20位）
- 确认密码（必填，需与密码一致）

**处理**：
- 验证用户名是否已存在
- 验证密码强度和一致性
- 默认创建为普通读者角色（isadmin=0）
- 将用户信息存储到数据库

**输出**：
- 注册成功：返回成功提示，跳转到登录页面
- 注册失败：返回错误信息（用户名已存在、密码不一致等）

#### 3.1.2 用户登录
**功能描述**：已注册用户通过用户名和密码登录系统。

**输入**：
- 用户名（必填）
- 密码（必填）
- 角色选择（管理员/读者）

**处理**：
- 验证用户名、密码和角色的匹配性
- 生成唯一的Token（有效期1小时）
- 将Token存储到Redis缓存

**输出**：
- 登录成功：返回Token和用户基本信息
- 登录失败：返回错误提示（用户名或密码错误、角色不匹配）

#### 3.1.3 用户信息查询
**功能描述**：根据Token获取当前登录用户的详细信息。

**输入**：Token

**处理**：
- 从Redis中验证Token有效性
- 查询用户详细信息

**输出**：
- 成功：返回用户信息（用户ID、用户名、角色）
- 失败：返回Token无效或已过期提示

#### 3.1.4 修改密码
**功能描述**：用户可以修改自己的登录密码。

**输入**：
- 用户ID
- 旧密码
- 新密码
- 确认新密码

**处理**：
- 验证旧密码是否正确
- 验证新密码和确认密码是否一致
- 更新数据库中的密码

**输出**：
- 成功：返回修改成功提示
- 失败：返回错误信息（旧密码错误、新密码不一致）

#### 3.1.5 用户管理（管理员功能）
**功能描述**：管理员可以查看、添加、修改、删除用户信息。

**查询功能**：
- 支持分页查询
- 支持按用户名模糊搜索
- 显示用户总数

**添加功能**：
- 输入：用户名、密码、角色
- 验证用户名唯一性
- 创建新用户记录

**修改功能**：
- 输入：用户ID、新的用户信息
- 更新用户记录

**删除功能**：
- 支持单个删除和批量删除
- 删除前需确认操作

### 3.2 图书信息管理模块

#### 3.2.1 图书信息查询
**功能描述**：用户可以查询图书信息，支持多条件搜索。

**输入**：
- 书名（可选，模糊匹配）
- 作者（可选，模糊匹配）
- 图书类型（可选，精确匹配）
- 分页参数（页码、每页数量）

**处理**：
- 根据搜索条件查询数据库
- 按分页参数返回结果

**输出**：
- 图书列表（包含书名、作者、价格、类型、借阅状态、封面图片等）
- 总记录数

#### 3.2.2 图书信息添加（管理员功能）
**功能描述**：管理员可以添加新的图书信息。

**输入**：
- 书名（必填）
- 作者（必填）
- 价格（必填，数值类型）
- 图书类型ID（必填）
- 图书描述（可选）
- 封面图片（可选）

**处理**：
- 验证必填字段
- 验证图书类型ID是否存在
- 默认设置借阅状态为"可借"（isborrowed=0）
- 插入图书记录到数据库

**输出**：
- 成功：返回添加成功提示
- 失败：返回错误信息

#### 3.2.3 图书信息修改（管理员功能）
**功能描述**：管理员可以修改图书的详细信息。

**输入**：
- 图书ID（必填）
- 需要修改的字段（书名、作者、价格、类型、描述、封面等）

**处理**：
- 验证图书ID是否存在
- 更新图书信息

**输出**：
- 成功：返回修改成功提示
- 失败：返回错误信息

#### 3.2.4 图书信息删除（管理员功能）
**功能描述**：管理员可以删除图书信息。

**输入**：
- 单个删除：图书ID
- 批量删除：图书ID数组

**处理**：
- 验证图书是否存在
- 检查图书是否处于借出状态（已借出的图书不允许删除）
- 删除图书记录

**输出**：
- 成功：返回删除成功提示
- 失败：返回错误信息（图书不存在、图书已借出）

#### 3.2.5 图书封面上传
**功能描述**：管理员可以为图书上传封面图片。

**输入**：
- 图片文件（支持jpg、png格式）
- 文件大小限制：10MB

**处理**：
- 验证文件格式和大小
- 为文件名添加时间戳防止重复
- 保存文件到服务器指定目录
- 返回图片的访问URL

**输出**：
- 成功：返回图片URL
- 失败：返回错误信息（格式不支持、文件过大）

#### 3.3.1 图书类型查询
**功能描述**：查询图书类型列表，所有用户均可访问。

**输入**：
- 分页参数（可选）

**处理**：
- 查询所有图书类型

**输出**：
- 类型列表（类型ID、类型名称、类型描述）

#### 3.3.2 图书类型管理（管理员功能）

**添加功能**：
- 输入：类型名称、类型描述
- 验证类型名称唯一性
- 创建新类型记录

**修改功能**：
- 输入：类型ID、新的类型信息
- 更新类型记录

**删除功能**：
- 输入：类型ID或类型ID数组
- 检查是否有图书使用该类型（有则不允许删除）

### 3.4 借阅管理模块

#### 3.4.1 借书功能
**功能描述**：读者可以借阅图书，系统自动记录借阅信息。

**输入**：
- 用户ID
- 图书ID

**处理**（事务操作）：
1. 验证用户和图书是否存在
2. 检查图书是否可借（isborrowed=0）
3. 更新图书状态为"已借出"（isborrowed=1）
4. 在借阅表中插入新记录（记录借阅时间）
5. 任何步骤失败则回滚事务

**输出**：
- 成功：返回借阅成功提示和借阅记录信息
- 失败：返回错误信息（图书不存在、图书已借出、库存不足）

**业务规则**：
- 同一本图书同一时间只能被一个读者借阅
- 借阅记录必须包含完整的用户和图书信息

#### 3.4.2 还书功能
**功能描述**：读者归还图书，系统更新借阅记录和图书状态。

**输入**：
- 借阅记录ID

**处理**（事务操作）：
1. 验证借阅记录是否存在
2. 检查是否已经还过书（returntime不为空）
3. 更新图书状态为"可借"（isborrowed=0）
4. 更新借阅记录的归还时间（returntime）
5. 任何步骤失败则回滚事务

**输出**：
- 成功：返回还书成功提示
- 失败：返回错误信息（记录不存在、已还过书）

#### 3.4.3 借阅记录查询
**功能描述**：查询借阅记录，支持多条件搜索。

**输入**：
- 用户ID（可选）
- 图书ID（可选）
- 分页参数

**处理**：
- 根据搜索条件查询借阅记录
- 支持分页返回

**输出**：
- 借阅记录列表（包含用户名、图书名、借阅时间、归还时间）
- 总记录数

#### 3.4.4 借阅记录管理（管理员功能）
**功能描述**：管理员可以添加、修改、删除借阅记录。

**添加功能**：
- 手动创建借阅记录（特殊情况使用）

**修改功能**：
- 修改借阅记录的详细信息

**删除功能**：
- 支持单个删除和批量删除
- 删除前需确认操作

## 4. 非功能需求

### 4.1 性能需求
- **响应时间**：普通查询操作响应时间不超过2秒
- **并发用户数**：系统应支持至少100个用户同时在线
- **数据库性能**：单表数据量达到10万条时，查询性能不明显下降
- **文件上传**：单个图片文件上传时间不超过5秒

### 4.2 安全性需求
- **身份认证**：所有操作必须通过Token验证
- **权限控制**：严格区分管理员和读者的访问权限
- **密码安全**：密码需加密存储（建议使用BCrypt或类似算法）
- **SQL注入防护**：使用参数化查询防止SQL注入攻击
- **XSS防护**：对用户输入进行过滤和转义
- **Token管理**：Token有效期1小时，过期后需重新登录

### 4.3 可靠性需求
- **数据一致性**：借阅和还书操作使用事务保证数据一致性
- **异常处理**：系统应能捕获并妥善处理各类异常
- **数据备份**：数据库应定期备份，防止数据丢失
- **系统可用性**：系统可用性应达到99%以上

### 4.4 可维护性需求
- **代码规范**：遵循Java编码规范和Spring Boot最佳实践
- **分层架构**：采用Controller-Service-Mapper三层架构
- **日志记录**：关键操作应记录日志，便于问题排查
- **文档完善**：提供完整的API文档和系统维护文档

### 4.5 可扩展性需求
- **模块化设计**：各功能模块低耦合，便于功能扩展
- **接口标准化**：采用RESTful API设计，便于前端集成
- **数据库设计**：预留扩展字段，便于功能升级
- **分布式支持**：使用Redis管理Session，支持分布式部署

### 4.6 兼容性需求
- **浏览器兼容**：支持主流浏览器（Chrome、Firefox、Safari、Edge）
- **移动端适配**：API设计考虑移动端访问需求
- **数据库兼容**：使用标准SQL语法，便于数据库迁移

## 5. 数据需求

### 5.1 数据实体

#### 5.1.1 用户（User）
| 字段名 | 类型 | 说明 | 约束 |
|--------|------|------|------|
| userid | INT | 用户ID | 主键，自增 |
| username | VARCHAR(50) | 用户名 | 非空，唯一 |
| userpassword | VARCHAR(100) | 密码 | 非空 |
| isadmin | TINYINT | 角色标识 | 1=管理员，0=读者 |

#### 5.1.2 图书信息（BookInfo）
| 字段名 | 类型 | 说明 | 约束 |
|--------|------|------|------|
| bookid | INT | 图书ID | 主键，自增 |
| bookname | VARCHAR(100) | 书名 | 非空 |
| bookauthor | VARCHAR(50) | 作者 | 非空 |
| bookprice | DECIMAL(10,2) | 价格 | 非空 |
| booktypeid | INT | 图书类型ID | 外键 |
| booktypename | VARCHAR(50) | 图书类型名称 | 冗余字段 |
| bookdesc | TEXT | 图书描述 | 可空 |
| isborrowed | TINYINT | 借阅状态 | 1=已借出，0=可借 |
| bookimg | VARCHAR(200) | 封面图片URL | 可空 |

#### 5.1.3 图书类型（BookType）
| 字段名 | 类型 | 说明 | 约束 |
|--------|------|------|------|
| booktypeid | INT | 类型ID | 主键，自增 |
| booktypename | VARCHAR(50) | 类型名称 | 非空，唯一 |
| booktypedesc | VARCHAR(200) | 类型描述 | 可空 |

#### 5.1.4 借阅记录（Borrow）
| 字段名 | 类型 | 说明 | 约束 |
|--------|------|------|------|
| borrowid | INT | 借阅记录ID | 主键，自增 |
| userid | INT | 用户ID | 外键 |
| username | VARCHAR(50) | 用户名 | 冗余字段 |
| bookid | INT | 图书ID | 外键 |
| bookname | VARCHAR(100) | 图书名称 | 冗余字段 |
| borrowtime | DATETIME | 借阅时间 | 非空 |
| returntime | DATETIME | 归还时间 | 可空，NULL表示未归还 |

### 5.2 数据关系
- book_info.booktypeid → book_type.booktypeid（多对一）
- borrow.userid → user.userid（多对一）
- borrow.bookid → book_info.bookid（多对一）

### 5.3 数据完整性约束
- 主键约束：所有表的主键字段不允许为空且唯一
- 外键约束：外键字段必须引用有效的主键值
- 唯一性约束：用户名、图书类型名称必须唯一
- 非空约束：关键字段不允许为空
- 默认值约束：isborrowed默认为0，borrowtime默认为当前时间

## 6. 接口需求

### 6.1 外部接口
- **前端接口**：提供RESTful API供前端调用
- **数据库接口**：通过MyBatis访问MySQL数据库
- **缓存接口**：通过Spring Data Redis访问Redis缓存

### 6.2 内部接口
- **Controller层**：接收HTTP请求，调用Service层
- **Service层**：实现业务逻辑，调用Mapper层
- **Mapper层**：执行数据库操作

### 6.3 接口规范
- **协议**：HTTP/HTTPS
- **数据格式**：JSON
- **字符编码**：UTF-8
- **响应格式**：统一的响应结构（包含状态码、消息、数据）
